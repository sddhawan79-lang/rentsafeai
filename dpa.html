means the Information Commissioner's Office (ICO).

4Nature & Purpose of Processing

The Processor carries out the following processing activities on behalf of the Controller:

  • Storing and organising documents uploaded by the Controller (tenancy agreements, certificates, identification documents, and similar);
  • Extracting structured data from documents using AI processing;
  • Storing and displaying property, tenancy, and compliance information entered by the Controller;
  • Generating compliance reminders, alerts, and deadline notifications based on data entered by the Controller;
  • Producing AI-generated summaries, compliance assessments, and document drafts based on Controller-provided data;
  • Transmitting transactional emails to addresses specified by the Controller;
  • Providing secure storage and access to uploaded documents.
Important limitation: All processing is purely administrative and organisational in nature. The Processor does not make legal decisions, does not determine compliance outcomes, and does not take autonomous action on the Controller's behalf. No automated decision-making within the meaning of Article 22 UK GDPR is performed.

Schedule 3 — Security Measures

The Processor implements the following technical and organisational security measures:

Technical Measures

  • Encryption in transit: All data transmitted between users and the Service is encrypted using TLS 1.2 or higher (HTTPS);
  • Encryption at rest: All data stored in the Supabase database is encrypted at rest using AES-256;
  • Document storage: Uploaded documents are stored in private, access-controlled Supabase Storage buckets — not publicly accessible;
  • Authentication: User access is controlled via Supabase Auth with hashed password storage; session tokens are used for authenticated access;
  • Access controls: Database access is restricted using Row-Level Security (RLS) policies ensuring users can only access their own data;
  • API security: All AI API calls are routed through a secure server-side edge function — API keys are never exposed client-side.

Organisational Measures

  • Access to production systems is restricted on a strict need-to-know basis;
  • Sub-processors are required to maintain equivalent security standards under written contract;
  • Security measures are reviewed periodically and updated in response to identified risks;
  • Personal Data Breaches are subject to the notification obligations in clause 14 of this DPA.
